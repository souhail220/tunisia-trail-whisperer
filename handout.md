# TrailMate Tunisia — Project Handout

A mobile-first web app for discovering and navigating Tunisia's outdoor trails, with AI-powered safety, navigation, and community features.

---

## 1. Tech Stack

- **Framework**: TanStack Start v1 (React 19, file-based routing, SSR-ready)
- **Build**: Vite 7
- **Styling**: Tailwind CSS v4 via `src/styles.css` (CSS variables, OKLCH color space)
- **UI Library**: shadcn/ui (Radix primitives) — `new-york` style
- **Icons**: lucide-react
- **Toasts**: sonner
- **Fonts**: Inter (400–800)
- **Analytics**: Lucky Orange tracking script (site-id `2eb07c85`)
- **Deployment target**: Cloudflare Workers (via `wrangler.jsonc`)

## 3. Routes & Features

### Core Flow

| Route | Purpose |
|---|---|
| `/` | 3-step onboarding (name, experience level, region) → persists to `localStorage` |
| `/explore` | Featured trip card with **Reserve now** (→ start hike), search, difficulty/region chips, map pins, guide bookings |
| `/routes` | Saved trails + recently AI-generated routes |
| `/generate` | Form-based AI route generator with results carousel |
| `/community` | Feed (like/comment/bookmark), hazard reports, share-trail FAB |
| `/ai-guide` | Chat assistant + simulated voice input |
| `/profile` | Stats, achievements, settings sheets, link to **/features** |
| `/trail/$id` | Static trail detail (save, share, start hike) |
| `/hike/$id` | Active hike: timer, distance, **Activate & interact** button after start |

### Advanced Features Hub (`/features`)

Grid of 12 self-contained modules accessible from Profile:

| Module | Description |
|---|---|
| **AI Safety Watch** | Sensor monitoring sim, fall/inactivity detection, SOS countdown |
| **Offline Emergency** | Decision-tree first-aid guide |
| **Mesh SOS Network** | Simulated Bluetooth peer-to-peer SOS relay |
| **AR Route Generator** | AI route + AR HUD compass overlay |
| **RadioMode Walkie-Talkie** | Push-to-talk channel selector |
| **Smart Gear Checklist** | AI packing list from difficulty/weather/budget |
| **AI Hiking Companion** | Live pace + safety coaching panel |
| **Enhanced Route Sharing** | GPX upload + geotagged hazard sharing |
| **GhostTrail Footprints** | Encrypted breadcrumbs with rescue key |
| **WildlifeID Lens** | Offline species recognition + journal |
| **StarPath Night Navigator** | Constellation compass for night nav |
| **ThermalRisk Scanner** | Heatstroke risk score (UV + temp + intensity) |

## 4. UI Design System

### 4.1 Design Philosophy
- **Mobile-first**: every screen lives inside `MobileShell` (phone-frame, bottom tab bar).
- **Soft, friendly outdoor brand**: rounded corners (1rem base radius), subtle shadows, generous whitespace.
- **Purple primary + green secondary** evokes nature + tech.
- **Token-driven**: no hardcoded colors in components — everything reads from CSS variables.

### 4.2 Color Tokens (`src/styles.css`)

| Token | Hex | Usage |
|---|---|---|
| `--primary` | `#534AB7` (purple) | CTAs, active state, brand |
| `--secondary` | `#3B6D11` (green) | Nature accents, success badges |
| `--warning` | `#EF9F27` (amber) | Caution, moderate difficulty |
| `--danger` / `--destructive` | `#E24B4A` (red) | SOS, hard difficulty, alerts |
| `--success` | green | Positive feedback |
| `--background` / `--card` | near-white | App canvas, surfaces |
| `--muted` / `--muted-foreground` | light grey / mid-grey | Secondary text |
| `--border` / `--input` | pale grey | Dividers, input outlines |

Defined in OKLCH for perceptual uniformity. Shadows: `--shadow-card` (subtle) and `--shadow-float` (purple-tinted, for floating CTAs).

### 4.3 Typography
- Font: **Inter** (sans-serif, weights 400→800)
- Hierarchy: page titles `text-2xl font-bold`, card titles `text-sm font-bold`, meta `text-[11px] text-muted-foreground`.

### 4.4 Radius Scale
Base `--radius: 1rem`. Components use `rounded-md`, `rounded-xl`, `rounded-2xl` for cards, `rounded-full` for chips/avatars.

## 5. Core Components

### 5.1 `MobileShell`
Wraps every page in a centered max-width phone container, adds the bottom navigation tabs (Explore, Routes, Generate, Community, Profile). Ensures consistent vertical scroll + safe-area padding.

### 5.2 `FeatureShell`
Used by all `/features/*` pages. Provides back arrow, page title, and content slot inside `MobileShell`.

### 5.3 `TrailCard`
Image-led card showing trail name, region, distance, elevation, and a difficulty badge (color-coded via tokens). Wrapped in `<Link to="/trail/$id">`.

### 5.4 `Button` (`components/ui/button.tsx`)
shadcn primitive with `cva` variants:
- **variant**: `default` (primary purple), `destructive` (red), `outline`, `secondary`, `ghost`, `link`
- **size**: `default` (h-9), `sm` (h-8), `lg` (h-10), `icon` (square)
- `asChild` prop lets the button render as `<Link>` while keeping styling.

### 5.5 `Input` (`components/ui/input.tsx`)
Standardized 36px-tall input: transparent background, subtle border, focus ring uses `--ring` (matches primary). Used for search, forms, onboarding name field.

### 5.6 `Card` family (`components/ui/card.tsx`)
Composable: `Card` (container, `rounded-xl border shadow`) + `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`, `CardFooter`. The app also uses raw `bg-card rounded-2xl shadow-[var(--shadow-card)]` blocks for denser layouts.

### 5.7 `Sheet`
Bottom/side drawer used for filters, settings menus, share-trail forms.

### 5.8 Other shadcn primitives in use
`badge`, `avatar`, `progress`, `slider`, `switch`, `tabs`, `dialog`, `dropdown-menu`, `sonner` (toasts), `carousel`, `select`, `textarea`.

## 6. Patterns & Conventions

- **State**: local `useState`; persistence via `localStorage` (`trailmate.profile`), SSR-guarded with `typeof window !== "undefined"`.
- **Navigation**: always `@tanstack/react-router` (`Link`, `useNavigate`), never `react-router-dom`. No trailing slashes.
- **Feedback**: `sonner` `toast()` for confirmations (booking, save, share).
- **Mock data**: all content from `src/lib/mock-data.ts` — no backend wired.
- **Icons**: `lucide-react`, sized via Tailwind `h-* w-*`.
- **No hardcoded colors** in JSX — only token classes (`bg-primary`, `text-muted-foreground`, etc.).

---

## 7. Global Wiring

- `__root.tsx` injects meta tags (OG, Twitter cards), the app stylesheet, and the Lucky Orange analytics script (async/defer).
- `<Toaster />` from sonner mounted once globally.
- Each route declares its own `head()` with unique `title` + `description` for SEO.

---

## 8. Out of Scope (intentionally)

- Real authentication / backend
- Real maps (placeholder map pins)
- Real AI inference (UI flows are wired to mock responses)
- Real sensor APIs (Safety Watch / ThermalRisk use simulated input)
